# 评论类型筛选设计

## 目标

在“任务详情”区域新增一个“评论类型”下拉筛选，支持按 `全部 / 书评 / 章评 / 段评` 过滤当前任务明细。

## 范围

- 只影响任务详情表格的前端展示结果。
- 与现有“搜索评论内容”关键词过滤叠加生效。
- 不改后端接口，不改任务列表，不改跑分逻辑。

## 方案

采用方案 1：在现有搜索框旁边新增一个下拉框。

原因：

- 改动最小，能复用现有前端本地过滤逻辑。
- 不需要调整表格结构，样式影响面最小。
- 用户操作路径清晰，和搜索入口放在同一组控制区。

## 交互说明

- 默认值为“全部”。
- 选择某个类型后，只展示 `row.comment_type` 等于该类型的行。
- 同时输入关键词时，结果需满足“类型匹配”和“关键词匹配”两个条件。
- 切换任务时保留当前筛选值。

## 验收标准

- 下拉框展示在任务详情操作区，紧邻搜索框。
- 选择“章评”时，表格中只显示章评数据。
- 切回“全部”时，表格恢复显示全部匹配搜索条件的数据。
- 与关键词搜索叠加使用时，结果正确。
