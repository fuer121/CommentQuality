# 子 Agent 协作协议

## 输入

子 Agent 启动前必须获得一个任务文件或明确任务段落。任务文件应包含：

- 任务 ID
- 背景文件
- 允许修改的文件范围
- 禁止修改的文件范围
- 验收命令
- 报告输出路径

## 输出

子 Agent 报告必须写入：

`project/agent-reports/<task-id>-<short-name>.md`

报告模板：

```md
# <task-id> 报告

## 结论

完成 / 未完成 / 阻塞

## 输入

- 使用的基线文件：
- 使用的代码文件：

## 变更

- 修改文件：
- 新增文件：

## 验证

- 命令：
- 结果：

## 风险

- 

## 建议下一步

- 
```

## 禁止事项

- 不得把聊天上下文当作需求依据。
- 不得修改与任务无关的文件。
- 不得删除用户或其他 Agent 的未确认变更。
- 不得提交 Git，除非任务明确授权。
