# Git 操作策略

## 当前状态

当前目录不是 Git 仓库。暂不自动初始化 Git，直到进入网站工程初始化阶段并明确代码结构。

## 初始化时机

满足以下条件后，由总控决定是否执行 `git init`：

- 网站工程结构已经确定。
- `.gitignore` 已包含 `.DS_Store`、`node_modules/`、`.env`、临时输出和导入样例中的敏感数据。
- 至少有一个可运行的最小工程或明确的基线文档需要版本化。

## 提交时机

推荐提交点：

- 基线提交：项目基线文件稳定后。
- 工程初始化提交：网站可启动且健康检查通过。
- 导入功能提交：导入、预览、校验通过。
- 跑分功能提交：Dify 调用和批量处理通过。
- 导出功能提交：结果展示和导出通过。
- 交付提交：端到端验证和经验沉淀完成。

## 推送与 PR 时机

只有存在远端仓库且当前分支有清晰交付边界时才推送。

只有满足以下条件才提交 PR：

- 功能阶段完成并通过验收。
- 无无关文件变更。
- PR 描述包含变更摘要、验证命令、风险和后续事项。

## 不提交内容

- `.env`、API Key、访问令牌。
- 用户导入的真实评论数据，除非已经脱敏并明确允许。
- 临时运行日志、大体积导出文件。
- `.DS_Store`。
